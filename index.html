WEEK 1

Pengenalan Komputasi Awan

Fondasi paling dasar yang wajib dikuasai sebelum topik apapun

📖 Definisi Cloud Computing Wajib Hafal
Definisi: Cloud computing adalah paradigma penyediaan sumber daya komputasi (server, storage, database, networking, software) melalui internet, dengan pembayaran sesuai penggunaan (pay-as-you-go), tanpa pengelolaan infrastruktur fisik secara langsung.

Kata kunci: internet · on-demand · pay-as-you-go · scalable · elastic

⚡ 5 Prinsip Cloud Computing (NIST) Wajib!
#PrinsipPenjelasan Sederhana
1On-demand self-serviceBisa langsung pakai sendiri, tanpa hubungi penyedia
2Broad network accessBisa diakses dari mana saja lewat internet
3Resource poolingSumber daya dipakai bersama banyak pengguna, tapi terisolasi
4Rapid elasticityKapasitas bisa naik-turun cepat sesuai kebutuhan
5Measured serviceBayar hanya sesuai yang dipakai (seperti listrik)
💡 Trik hafal: O-B-R-R-M = On-demand · Broad · Resource pooling · Rapid · Measured
🏢 Jenis Deployment Cloud Penting
JenisSiapa yang pakaiContoh
Public CloudUmum, siapa sajaAWS, GCP, Azure
Private CloudSatu organisasi sajaServer cloud bank, rumah sakit
Hybrid CloudKombinasi keduanyaHSBC — data sensitif di private, analitik di public
Community CloudKelompok organisasi sejenisCloud khusus instansi pemerintah
💡 Analogi Sederhana
Analogi Listrik PLN: Dulu, pabrik harus punya generator sendiri. Sekarang cukup colok ke PLN dan bayar per bulan sesuai pemakaian. Cloud computing = "PLN untuk komputer". Kamu tidak perlu beli server fisik, cukup "colok ke cloud" dan bayar sesuai yang dipakai.

Model Layanan Cloud: IaaS, PaaS, SaaS

Topik paling sering keluar di UTS — kuasai ini dengan sangat baik

⚖️ Perbandingan 3 Model Layanan Sangat Wajib!
AspekIaaSPaaSSaaS
KepanjanganInfrastructure as a ServicePlatform as a ServiceSoftware as a Service
Yang dikelola userOS, Aplikasi, Data, Runtime, MiddlewareAplikasi, DataTidak ada (hanya pakai)
Yang dikelola providerVirtualization, Server, Storage, NetworkingRuntime, Middleware, OS, Virtualization, Server, Storage, NetworkingSemuanya
Kontrol userTinggiSedangRendah
Kemudahan pakaiSulit (butuh keahlian teknis)SedangMudah
Target penggunaSystem Admin, DevOpsDeveloperEnd user biasa
ContohAWS EC2, GCP Compute Engine, Azure VMsGoogle App Engine, Heroku, AWS Elastic BeanstalkGmail, Zoom, Google Docs, Dropbox
AnalogiSewa tanah kosong, bangun sendiriSewa rumah sudah ada dinding & atap, isi sendiriSewa apartemen furnished — langsung tinggal
✅ Kapan Menggunakan Model Apa?
Gunakan IaaS → Ketika butuh kontrol penuh atas sistem: konfigurasi server, OS, jaringan. Cocok untuk perusahaan dengan tim IT besar.
Gunakan PaaS → Ketika hanya fokus pada pengembangan aplikasi. Developer tidak mau pusing urusan server. Cocok untuk startup yang ingin bergerak cepat.
Gunakan SaaS → Ketika hanya butuh aplikasi siap pakai. Tidak ada tim IT. Contoh: perusahaan kecil pakai Gmail dan Zoom.
🧠 Rumus Pola Berpikir Model Layanan
Formula: Semakin TINGGI level layanan (IaaS → PaaS → SaaS) = Semakin SEDIKIT kontrol user + Semakin MUDAH dipakai + Semakin BANYAK yang dikelola provider
Cara ingat tanggung jawab: Di IaaS, user kelola dari OS ke atas. Di PaaS, user kelola dari Aplikasi ke atas. Di SaaS, user tidak kelola apapun.

WEEK 4

Virtualisasi

Teknologi inti yang membuat cloud computing bisa bekerja

🖥️ Apa itu Virtualisasi? Wajib Paham
Definisi akademis: Virtualisasi adalah teknologi yang memungkinkan satu perangkat keras fisik (hardware) dibagi menjadi beberapa unit komputasi virtual yang terisolasi satu sama lain, menggunakan lapisan perangkat lunak yang disebut Hypervisor.
Analogi: Bayangkan satu tanah luas (server fisik) dibagi menjadi beberapa apartemen (VM) dengan tembok (isolasi). Tiap penghuni tidak tahu penghuni lain ada. Pengelola gedung = Hypervisor.
📦 Jenis-Jenis Virtualisasi
JenisApa yang divirtualisasiContoh
Server VirtualizationServer fisik → beberapa VMVMware ESXi, Hyper-V
Storage VirtualizationBanyak storage → tampak satu unitAWS S3, RAID
Network VirtualizationJaringan fisik → Virtual NetworkVLAN, VPC (Virtual Private Cloud)
Desktop VirtualizationDesktop OS berjalan di serverVDI (Virtual Desktop Infrastructure)
Application VirtualizationApp berjalan terisolasi tanpa instalasiDocker Container
⚙️ Jenis Hypervisor Sering Keluar
TipeNamaCara KerjaContoh
Type 1Bare-metal HypervisorLangsung di atas hardware, tanpa OS hostVMware ESXi, Microsoft Hyper-V, Xen
Type 2Hosted HypervisorDi atas OS host (terinstall seperti aplikasi biasa)VirtualBox, VMware Workstation
Cara bedain: Type 1 = LANGSUNG di hardware (performa lebih baik, dipakai di data center). Type 2 = di ATAS OS (mudah diinstal, dipakai di laptop personal).

AspekMonolithMicroservices
StrukturSatu aplikasi besar terintegrasiBanyak layanan kecil independen

🔑 Konsep Penting Arsitektur Cloud
Load Balancer: Mendistribusikan traffic ke beberapa server agar tidak ada server yang overload. Seperti kasir yang mengarahkan antrian ke loket yang kosong.
Auto Scaling: Sistem otomatis tambah atau kurangi jumlah server berdasarkan beban. Saat banyak pengguna → tambah server. Saat sepi → kurangi server.
CDN (Content Delivery Network): Jaringan server yang tersebar di banyak lokasi geografis untuk menyajikan konten dari server terdekat ke pengguna. Tujuan: kurangi latency.
High Availability: Sistem dirancang agar tetap beroperasi meski ada komponen yang gagal. Gunakan redundancy (duplikasi komponen penting).

Komponen fisik dan logis yang membangun layanan cloud

KomponenFungsiDi cloud, setara dengan

StorageSimpan dataObject Storage (S3), Block Storage (EBS)
NetworkingKoneksi antar komponenVPC, Subnet, Load Balancer
DatabaseKelola data terstrukturAWS RDS, Cloud SQL, Azure DB

💾 Jenis Storage Cloud
JenisCara KerjaCocok untukContoh
Object StorageData disimpan sebagai objek dengan metadataFile, gambar, video, backupAWS S3, GCS
Block StorageData dipecah jadi blok-blok kecilDatabase, OS, aplikasi performa tinggiAWS EBS, Azure Disk
File StorageHierarki folder seperti file system biasaBerbagi file antar VMAWS EFS, Azure Files
🌐 Konsep Networking di Cloud
VPC (Virtual Private Cloud): Jaringan virtual private yang terisolasi di dalam cloud public. Pengguna bisa definisikan IP range, subnet, routing, dan firewall sendiri.
Subnet: Pembagian jaringan dalam VPC. Public subnet = bisa diakses internet. Private subnet = hanya akses internal.
Security Group / Firewall: Aturan yang mengontrol traffic masuk (inbound) dan keluar (outbound) dari VM/instance.

Webserver on Cloud

Implementasi dasar: menjalankan web server di infrastruktur cloud

🌐 Konsep Dasar Web Server di Cloud
Web server di cloud berarti menjalankan software web server (seperti Apache atau Nginx) di dalam Virtual Machine yang disewa dari cloud provider (seperti AWS EC2), sehingga website bisa diakses melalui internet.
🚀 Alur Deploy Web Server di Cloud
LangkahTindakanContoh di AWS
1Buat VM/InstanceLaunch EC2 Instance (pilih OS: Ubuntu)
2Konfigurasi keamananSecurity Group: buka port 80 (HTTP) dan 22 (SSH)
3Koneksi ke VMSSH ke instance menggunakan key pair
4Install web serverapt install apache2 / nginx
5Upload konten webCopy file HTML ke /var/www/html/
6Akses via IP publikBuka browser → masukkan IP publik EC2
⚙️ Perbandingan Apache vs Nginx
AspekApacheNginx
ModelProcess-based (fork per koneksi)Event-driven (asinkron)
Performa traffic tinggiKurang efisienLebih efisien
KonfigurasiPer-direktori (.htaccess)Terpusat
Umum digunakanWeb server aplikasi PHPReverse proxy, load balancer, web server statis
🌍 IP Publik vs IP Privat
IP Publik: Alamat IP yang bisa diakses dari internet. VM di cloud mendapat IP publik agar website bisa diakses pengguna luar.
IP Privat: Alamat IP internal dalam jaringan VPC. Tidak bisa diakses langsung dari internet. Dipakai untuk komunikasi antar VM di dalam cloud.

⚠️ Kesalahan Umum Mahasiswa
❌ Salah bedain IaaS dan PaaS: Ingat, di PaaS developer tidak urus server sama sekali. Di IaaS masih urus OS.
❌ Menganggap serverless = tidak ada server: Server tetap ada, tapi dikelola provider. Developer hanya tulis kode fungsi.
❌ Mengira container = virtual machine: Container tidak punya OS sendiri, jauh lebih ringan dari VM.
❌ Lupa contoh nyata: Soal UTS sering minta contoh. Hafal minimal 2 contoh per model layanan.
❌ Salah definisi Hypervisor: Hypervisor adalah SOFTWARE yang memungkinkan virtualisasi, bukan hardware.
✏️ Tips Menjawab Soal Teori Cloud Computing
1
Definisi dulu
Mulai jawaban dengan definisi konsep yang ditanya. Ini menunjukkan pemahaman dasar.
2
Karakteristik
Sebutkan karakteristik/ciri-ciri utama secara terstruktur (poin-poin).
3
Contoh nyata
Berikan contoh konkret (nama perusahaan/produk). Ini nilai tambah besar.
4
Kaitkan manfaat
Jelaskan kenapa konsep ini penting / apa manfaatnya bagi bisnis atau teknologi.
5
Analisis skenario
Kalau soal analisis: baca skenario, identifikasi kebutuhan, cocokkan dengan konsep yang tepat.
6
Hindari hafalan buta
Pahami konsep, jangan hafal kata per kata. Soal UTS biasanya variasi dari konsep dasar.
